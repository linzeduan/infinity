## STEP 2 · 决策原则提炼

**原则一：科技是市场命门——科技崩，市场崩**
- 触发信号：科技板块出现滞涨、冲高回落、高位浮盈筹码松动
- 判断逻辑：当前市场核心驱动力是科技抱团行情，科技板块一跌，其他板块因资金调仓被抽血式跟跌
- 对应操作：密切监控科技板块动向，科技走弱时整体仓位要收缩
- 反例边界：若市场驱动力从科技切换至其他主线（如消费、周期），科技对指数的拖拽效应会减弱

**原则二：跳水是获利盘兑现，不是行情终结**
- 触发信号：自3700点到4200点累积大量获利盘后出现的集中抛售
- 判断逻辑：此次跳水是区间内正常的获利回吐，而非趋势性反转——全年行情仍看好
- 对应操作：大跌后可小仓位博反弹，但不重仓追涨
- 反例边界：若跌破3800点且无有效反弹，可能从"区间震荡"转为"趋势性下跌"，此原则失效

**原则三：高波动是一切资金的敌人**
- 触发信号：市场进入4000-4200宽幅震荡区间，日内波动剧烈
- 判断逻辑：高波动环境下，追涨杀跌的损耗远超趋势行情，"大A不会让你舒服赚钱"
- 对应操作：仓位控制优先，留有子弹应对后续可能的深度调整
- 反例边界：若市场从高波动转为单边趋势（无论涨跌），此原则需切换为趋势跟踪策略

**原则四：中长期信心锚定资本开支增速，不因短期调整动摇**
- 触发信号：AI行情阶段性休整，短期回调引发恐慌
- 判断逻辑：只要资本开支增速不出现明显回落，AI产业趋势仍在→短期调整不改中长期逻辑
- 对应操作：不因短期下跌清仓科技底仓，保留核心仓位
- 反例边界：若科技企业资本开支增速显著放缓（如季报显示capex同比下降），中长期逻辑需重新评估

## STEP 3 · 持仓管理逻辑

```
科技板块：
  滞涨 + 高位浮盈筹码松动 + 冲高回落频繁 → 不追涨
  大跌后 → 可小仓位博反弹（非重仓）
  仓位控制 → 万一深度调整，留有子弹

指数整体：
  4000-4200区间震荡 → 高波动，不追涨杀跌
  大概率跌破4000 → 等跌破后再评估
  悲观触及3800 → 需观察是否止跌，不盲目抄底
  全年看好4400-4500 → 深跌是配置机会而非逃跑信号

操作纪律：
  首先 → 不追涨
  其次 → 大跌可小仓位博反弹
  最后 → 仓位控制，留有子弹
```

## STEP 4 · 风险意识画像

**最敏感的风险类型**：追涨风险 + 仓位风险
- "高波动是一切资金的敌人"——对高波动环境下的追涨损耗高度警惕
- "万一深度调整，后续也留有子弹"——始终为最坏情形预留弹药

**规避方式**：
- 不追涨（第一条纪律）
- 大跌才出手，且仅小仓位博反弹
- 仓位控制，不满仓操作

**持有期望**：中短期操作为主
- 有明确的指数区间判断（3800/4000/4200/4400-4500）
- 在区间内做波段操作，不恋战
- 全年看好但不意味着一直持有，而是"跌了买、涨了卖"
