## 一、商业模式

### 001\. 这个赛道还能不能做？

1. **原问题**
	：这个赛道还能不能做？
2. **错在哪里**
	：把「赛道」当成能决定赚钱的实体，但赚钱的是具体交易结构，不是赛道
3. **隐含假设**
	：选对赛道就能赚钱，选错赛道就不能赚钱
4. **大白话改写**
	：我想做的这个具体产品，有没有人愿意持续付钱？
5. **真正该做的事**
	：列出买家、产品、价格、获客、转化、交付、复购，再找 3 个正在赚钱的对标
6. **一句话**
	：不要问赛道，问谁付钱、为什么付、付多少

### 002\. 我适不适合创业？

1. **原问题**
	：我适不适合创业？
2. **错在哪里**
	：「适合」没有定义。它可以指性格、资源、能力、风险承受力，也可以只是害怕开始
3. **隐含假设**
	：创业前存在一个可以提前判定的「适合度」
4. **大白话改写**
	：我能不能用现有资源完成一个最小交易？
5. **真正该做的事**
	：用 7 天卖出一个最小产品。卖不出去再分析哪里不行
6. **一句话**
	：创业不是性格测试，是交易测试

### 003\. 我的商业模式行不行？

1. **原问题**
	：我的商业模式行不行？
2. **错在哪里**
	：信息太少。「行」也没有标准，是能活、能赚、能规模化，还是能融资？
3. **隐含假设**
	：商业模式可以脱离数据被一句话判断
4. **大白话改写**
	：我的获客、转化、交付、复购和利润结构有没有闭环？
5. **真正该做的事**
	：把最近 30 天的流量、线索、成交、客单、毛利、交付成本列出来
6. **一句话**
	：没有数字的商业模式，只是商业想象

### 004\. 我想创业但没有钱怎么办？

1. **原问题**
	：我想创业但没有钱怎么办？
2. **错在哪里**
	：把钱当成创业前提，但很多早期生意最需要的不是钱，是真实需求和成交能力
3. **隐含假设**
	：没有启动资金就不能开始
4. **大白话改写**
	：有没有一个不花钱也能验证的最小交易？
5. **真正该做的事**
	：先做服务、咨询、代运营、撮合、预售，找到能收钱的需求
6. **一句话**
	：没钱不是不能创业，是不能玩烧钱游戏

### 005\. 我有资源，怎么变现？

1. **原问题**
	：我有资源，怎么变现？
2. **错在哪里**
	：「资源」太模糊。资源不是资产，能被买家付费调用的东西才有商业意义
3. **隐含假设**
	：只要我拥有某种资源，它天然就能卖钱
4. **大白话改写**
	：我手里的东西能帮谁解决什么问题，谁愿意为它付多少钱？
5. **真正该做的事**
	：把资源翻译成买家的收益、节省、机会或风险降低
6. **一句话**
	：不能被买家使用的资源，不叫资源，叫库存

### 006\. 我想做高客单价业务，怎么做？

1. **原问题**
	：我想做高客单价业务，怎么做？
2. **错在哪里**
	：把高客单价当成目标，但高价格来自高价值、高信任和高风险承接
3. **隐含假设**
	：把价格调高就变成高客单价业务
4. **大白话改写**
	：我能不能解决一个值得客户多付钱的问题？
5. **真正该做的事**
	：先定义高价值场景，再设计交付、保障、案例和筛选机制
6. **一句话**
	：高客单价不是贵，是客户觉得不买更贵

### 007\. 我应该做 To B 还是 To C？

1. **原问题**
	：我应该做 To B 还是 To C？
2. **错在哪里**
	：分类太粗。To B 和 To C 不是答案，具体买家、决策链和支付理由才是答案
3. **隐含假设**
	：商业方向可以用客户类型二分法决定
4. **大白话改写**
	：我更容易让哪一类具体买家完成付款？
5. **真正该做的事**
	：分别画出两个方向的获客成本、成交周期、客单价、交付成本、复购可能
6. **一句话**
	：不要选 B 或 C，选更容易收钱的那条链

### 008\. 我这个项目有没有前景？

1. **原问题**
	：我这个项目有没有前景？
2. **错在哪里**
	：「前景」是远处的雾。早期项目先问能不能成交，不问十年后是不是宏大叙事
3. **隐含假设**
	：有前景的项目现在就值得做
4. **大白话改写**
	：这个项目现在有没有真实买家和真实利润？
5. **真正该做的事**
	：先做 10 个陌生成交或 3 个复购，再谈前景
6. **一句话**
	：前景不能付款，客户能付款

### 009\. 我应该先做产品还是先做流量？

1. **原问题**
	：我应该先做产品还是先做流量？
2. **错在哪里**
	：把产品和流量当成先后关系，但更底层的问题是你有没有交易闭环
3. **隐含假设**
	：产品和流量可以脱离成交单独优化
4. **大白话改写**
	：我现在缺的是可卖的东西，还是缺把买家带过来的路径？
5. **真正该做的事**
	：先做一个可付款的最小产品，再用最便宜的渠道测试转化
6. **一句话**
	：没有产品的流量是热闹，没有流量的产品是自嗨

### 010\. 我的业务怎么规模化？

1. **原问题**
	：我的业务怎么规模化？
2. **错在哪里**
	：很多业务还没证明能稳定赚钱，就开始幻想规模化
3. **隐含假设**
	：规模化是所有业务的下一步
4. **大白话改写**
	：我现在的赚钱动作能不能被重复、复制、交给别人执行？
5. **真正该做的事**
	：先把获客、转化、交付做成 SOP，计算每复制一次是否增加利润
6. **一句话**
	：不能稳定重复的赚钱动作，放大以后只会稳定放大混乱

## 二、内容创作

### 011\. 我的内容为什么没有流量？

1. **原问题**
	：我的内容为什么没有流量？
2. **错在哪里**
	：把「没有流量」当成单一原因，但可能是选题、标题、封面、表达、平台、样本量的问题
3. **隐含假设**
	：流量失败有一个简单原因
4. **大白话改写**
	：用户为什么要在信息流里停下来，看完并互动？
5. **真正该做的事**
	：拆 20 条同类爆款，对比选题冲突、标题承诺、封面信息、开头密度
6. **一句话**
	：流量不是平台欠你的，是内容没有抢到注意力

### 012\. 我应该做什么内容？

1. **原问题**
	：我应该做什么内容？
2. **错在哪里**
	：「应该」没有标准。是为了涨粉、成交、建立信任，还是训练表达？
3. **隐含假设**
	：存在一个适合所有人的内容方向
4. **大白话改写**
	：我想通过内容让谁产生什么行动？
5. **真正该做的事**
	：先确定产品和买家，再倒推内容选题
6. **一句话**
	：内容不是为了表达自己，是为了改变用户的下一步动作

### 013\. 我的内容 AI 味太重怎么办？

1. **原问题**
	：我的内容 AI 味太重怎么办？
2. **错在哪里**
	：把问题归因给 AI，但真正问题通常是人没有文字洁癖
3. **隐含假设**
	：AI 写的内容天然有问题
4. **大白话改写**
	：这段文字是不是空、假、滑、套、没有具体判断？
5. **真正该做的事**
	：删掉套话、排比、抽象名词，补具体对象、冲突、判断和例子
6. **一句话**
	：不是 AI 味，是你允许烂文字过稿

### 014\. 我要不要追热点？

1. **原问题**
	：我要不要追热点？
2. **错在哪里**
	：热点不是目的。问题是热点能不能承载你的判断、产品和用户关系
3. **隐含假设**
	：热点会自动带来有价值的流量
4. **大白话改写**
	：这个热点能不能让我说出和业务有关的有效判断？
5. **真正该做的事**
	：只追能连接你的产品、方法论、案例库和买家痛点的热点
6. **一句话**
	：追不上热点不可怕，被热点带着跑才可怕

### 015\. 我是做短视频还是公众号？

1. **原问题**
	：我是做短视频还是公众号？
2. **错在哪里**
	：平台不是第一问题，内容形态和成交路径才是第一问题
3. **隐含假设**
	：选对平台就能解决内容问题
4. **大白话改写**
	：我的选题更适合被快速感知，还是适合被深度阅读？
5. **真正该做的事**
	：观点冲突做短视频，深逻辑做公众号，工具清单做图文，再看转化路径
6. **一句话**
	：平台不是家，是分发工具

### 016\. 我的标题怎么写更吸引人？

1. **原问题**
	：我的标题怎么写更吸引人？
2. **错在哪里**
	：标题不能替一个无价值选题打工
3. **隐含假设**
	：标题技巧能拯救内容本身
4. **大白话改写**
	：这条内容到底给用户一个什么强承诺？
5. **真正该做的事**
	：先把选题改到有冲突、有收益、有对象，再写标题
6. **一句话**
	：标题不是化妆，是把内容里最硬的东西露出来

### 017\. 我想做干货内容，可以吗？

1. **原问题**
	：我想做干货内容，可以吗？
2. **错在哪里**
	：「干货」是伪概念。用户要的不是干，是解决问题、获得判断、减少试错
3. **隐含假设**
	：信息密度高就是好内容
4. **大白话改写**
	：我的内容能不能让用户少走一步错路？
5. **真正该做的事**
	：用案例、判断标准、反例和行动步骤替代信息堆砌
6. **一句话**
	：干货不是把知识压缩，是把问题讲清楚

### 018\. 我内容很专业，为什么没人看？

1. **原问题**
	：我内容很专业，为什么没人看？
2. **错在哪里**
	：专业是你的感受，不是用户的阅读理由
3. **隐含假设**
	：专业会自动产生注意力
4. **大白话改写**
	：用户看完能不能立刻理解这和他有什么关系？
5. **真正该做的事**
	：把专业语言翻译成场景、痛点、损失、收益和选择
6. **一句话**
	：专业不是让人觉得你厉害，是让人觉得自己被说中了

### 019\. 我想打造个人 IP，怎么定位？

1. **原问题**
	：我想打造个人 IP，怎么定位？
2. **错在哪里**
	：「IP 定位」经常是假问题。别人记住你，不是因为你写了定位句
3. **隐含假设**
	：存在一个可设计的正确定位
4. **大白话改写**
	：我能长期用什么判断帮助哪类人？
5. **真正该做的事**
	：连续输出 30 条同一类问题的高质量判断，让外界形成记忆
6. **一句话**
	：定位不是你怎么介绍自己，是别人反复因为什么想起你

### 020\. 我每天发，为什么还没涨粉？

1. **原问题**
	：我每天发，为什么还没涨粉？
2. **错在哪里**
	：把频率当因果。持续发和涨粉相关，但不必然因果
3. **隐含假设**
	：只要坚持发布，平台就应该奖励我
4. **大白话改写**
	：我每天发的东西，有没有每天变得更值得看？
5. **真正该做的事**
	：停止机械打卡，建立复盘表：选题、开头、完播、互动、转粉原因
6. **一句话**
	：坚持错误动作，只会稳定生产错误结果

## 三、AI 工具 / Agent

### 021\. 我想做一个 AI Agent，怎么变现？

1. **原问题**
	：我想做一个 AI Agent，怎么变现？
2. **错在哪里**
	：Agent 是形式，不是产品。变现的是被解决的问题
3. **隐含假设**
	：只要是 Agent，就天然有商业价值
4. **大白话改写**
	：这个 Agent 替谁完成什么高频、高价值、可验证的任务？
5. **真正该做的事**
	：先找人工服务中重复、昂贵、规则明确的一段流程，再做 Agent
6. **一句话**
	：不要卖 Agent，卖被 Agent 省掉的麻烦

### 022\. 我应该学提示词还是学编程？

1. **原问题**
	：我应该学提示词还是学编程？
2. **错在哪里**
	：把工具技能当成方向选择，跳过了要解决的问题
3. **隐含假设**
	：技能选择本身会决定商业结果
4. **大白话改写**
	：我现在要自动化的任务，需要语言控制，还是需要系统集成？
5. **真正该做的事**
	：先定义一个工作流，再判断缺的是提示词、脚本、API 还是产品设计
6. **一句话**
	：别问学什么，问你要让机器替你做什么

### 023\. AI 会不会替代我的工作？

1. **原问题**
	：AI 会不会替代我的工作？
2. **错在哪里**
	：问题太大。AI 不替代职业名称，替代具体任务
3. **隐含假设**
	：工作是一个整体，要么被替代，要么不被替代
4. **大白话改写**
	：我工作里的哪些任务可以被 AI 更便宜、更快、更稳定地完成？
5. **真正该做的事**
	：把工作拆成任务清单，标注判断、沟通、执行、审美、责任归属
6. **一句话**
	：AI 不替代「你」，AI 替代你那些说不清楚但还在重复做的动作

### 024\. 我想做 AI 工具，选什么方向？

1. **原问题**
	：我想做 AI 工具，选什么方向？
2. **错在哪里**
	：从技术出发，而不是从付费场景出发
3. **隐含假设**
	：方向来自模型能力，而不是用户愿意付钱的痛
4. **大白话改写**
	：哪类人现在已经在为某个低效流程付钱？
5. **真正该做的事**
	：找 10 个正在购买人工服务、SaaS 或课程的人，问他们哪一步最痛
6. **一句话**
	：AI 工具不是展示模型能力，是接管一段付费流程

### 025\. 我的 Agent 为什么没人用？

1. **原问题**
	：我的 Agent 为什么没人用？
2. **错在哪里**
	：没人用不一定是功能差，可能是任务不刚、入口不对、信任不足、结果不可验
3. **隐含假设**
	：用户看到自动化就会想用
4. **大白话改写**
	：用户现在有没有一个必须完成、并且愿意交给工具的任务？
5. **真正该做的事**
	：观察用户不用你的 Agent 时怎么完成任务，找出真实替代品
6. **一句话**
	：没人用，通常不是不懂 AI，是没有非用不可的场景

### 026\. AI 写作还能不能做？

1. **原问题**
	：AI 写作还能不能做？
2. **错在哪里**
	：「AI 写作」太粗。写文章、写销售页、写脚本、写报告是完全不同的业务
3. **隐含假设**
	：一个技术标签可以定义一个市场
4. **大白话改写**
	：哪种写作任务有明确交付标准和付费意愿？
5. **真正该做的事**
	：选一个垂直场景，做输入模板、审核标准、案例库和人工兜底
6. **一句话**
	：AI 写作不值钱，特定场景的合格交付值钱

### 027\. 我需要做自己的知识库吗？

1. **原问题**
	：我需要做自己的知识库吗？
2. **错在哪里**
	：知识库不是收藏夹。不能进入工作流的知识库只是囤积癖
3. **隐含假设**
	：资料越多，AI 越聪明
4. **大白话改写**
	：这些材料能不能帮助 AI 在具体任务中做出更好的判断？
5. **真正该做的事**
	：围绕任务建立案例、规则、反例、术语表，而不是乱塞文档
6. **一句话**
	：知识库不是仓库，是给任务供血的规则系统

### 028\. 我怎么把经验做成 Skill？

1. **原问题**
	：我怎么把经验做成 Skill？
2. **错在哪里**
	：经验不能直接变 Skill，必须先显性化成判断标准和流程
3. **隐含假设**
	：把经验写进去，AI 就会理解
4. **大白话改写**
	：我做这件事时到底按什么顺序判断，遇到什么信号怎么处理？
5. **真正该做的事**
	：整理触发条件、输入要求、判断流程、输出格式、案例和禁区
6. **一句话**
	：Skill 不是经验描述，是把隐性判断改造成机器可执行规则

### 029\. 我做的 AI 产品需要多强的模型？

1. **原问题**
	：我做的 AI 产品需要多强的模型？
2. **错在哪里**
	：模型强弱不是第一问题，任务边界、数据质量和容错机制更重要
3. **隐含假设**
	：更强模型会自动解决产品问题
4. **大白话改写**
	：这个任务允许多大错误，错误后谁负责，怎么纠正？
5. **真正该做的事**
	：先定义任务难度、验收标准、失败成本，再选模型
6. **一句话**
	：模型能力不是产品能力，系统设计才是产品能力

### 030\. 我能不能用 AI 做一个全自动赚钱机器？

1. **原问题**
	：我能不能用 AI 做一个全自动赚钱机器？
2. **错在哪里**
	：把商业复杂性幻想成自动化问题
3. **隐含假设**
	：赚钱可以绕过买家、信任、交付和责任
4. **大白话改写**
	：我能不能用 AI 降低某个赚钱流程里的成本或提高转化？
5. **真正该做的事**
	：选一个已有收入的流程，用 AI 替换其中一段，而不是凭空发明印钞机
6. **一句话**
	：AI 能放大商业模式，不能替你凭空制造商业模式

## 四、知识付费

### 031\. 我做知识付费会不会割韭菜？

1. **原问题**
	：我做知识付费会不会割韭菜？
2. **错在哪里**
	：把道德焦虑放在交易结构之前。关键不是知识付费，而是承诺和交付是否匹配
3. **隐含假设**
	：知识付费天然可疑
4. **大白话改写**
	：我收的钱和我能交付的结果是否匹配？
5. **真正该做的事**
	：写清楚适合谁、不适合谁、能得到什么、不能保证什么
6. **一句话**
	：割不割韭菜，不看品类，看你有没有用假承诺收真钱

### 032\. 我的课应该卖多少钱？

1. **原问题**
	：我的课应该卖多少钱？
2. **错在哪里**
	：价格不能凭感觉定，定价就是产品设计
3. **隐含假设**
	：价格是内容成本加一点利润
4. **大白话改写**
	：这个课解决的问题，对用户值多少钱？
5. **真正该做的事**
	：看对标、测预售、区分引流款和利润款，设计不同交付层级
6. **一句话**
	：价格不是数字，是筛选客户和定义产品的工具

### 033\. 我没有很厉害，可以卖课吗？

1. **原问题**
	：我没有很厉害，可以卖课吗？
2. **错在哪里**
	：把「厉害」当资格，但用户买的是比自己更清楚的一段路径
3. **隐含假设**
	：只有顶级专家才有资格收费
4. **大白话改写**
	：我能不能帮某类人解决一个比我低一阶的问题？
5. **真正该做的事**
	：只卖自己验证过、能交付、边界清楚的内容
6. **一句话**
	：不是你够不够厉害，是你有没有资格承诺这个结果

### 034\. 我应该做训练营还是录播课？

1. **原问题**
	：我应该做训练营还是录播课？
2. **错在哪里**
	：形式选择不能脱离用户任务。不同问题需要不同交付密度
3. **隐含假设**
	：产品形态有天然优劣
4. **大白话改写**
	：用户完成这件事，需要信息、反馈、陪跑，还是监督？
5. **真正该做的事**
	：简单知识做录播，需要反馈和行为改变的做训练营
6. **一句话**
	：交付形式由用户改变难度决定，不由你的制作偏好决定

### 035\. 我的知识很零散，怎么做成产品？

1. **原问题**
	：我的知识很零散，怎么做成产品？
2. **错在哪里**
	：知识不是产品，解决路径才是产品
3. **隐含假设**
	：把知识整理好就能卖
4. **大白话改写**
	：用户从 A 状态到 B 状态，中间需要哪些判断和动作？
5. **真正该做的事**
	：按用户任务重组知识，设计输入、步骤、练习、反馈和验收
6. **一句话**
	：知识产品卖的不是知识，是一条更少踩坑的路径

### 036\. 我怕学员没结果怎么办？

1. **原问题**
	：我怕学员没结果怎么办？
2. **错在哪里**
	：结果不是单方控制。你能承诺的是交付、反馈、边界和筛选
3. **隐含假设**
	：卖课必须保证所有人成功
4. **大白话改写**
	：哪些结果由我负责，哪些结果由学员行动负责？
5. **真正该做的事**
	：设置报名门槛、作业机制、适用人群和不适用人群
6. **一句话**
	：你不能替学员行动，但你必须对自己的承诺负责

### 037\. 我应该先免费分享还是直接收费？

1. **原问题**
	：我应该先免费分享还是直接收费？
2. **错在哪里**
	：免费和收费不是道德选择，是信任和产品阶段选择
3. **隐含假设**
	：免费会自动带来付费，或者收费会吓跑用户
4. **大白话改写**
	：用户现在对我有多少信任，对这个问题有多强付费意愿？
5. **真正该做的事**
	：免费内容负责证明判断力，付费产品负责交付完整路径
6. **一句话**
	：免费不是慈善，收费不是冒犯，它们都是交易设计

### 038\. 我的知识付费怎么做复购？

1. **原问题**
	：我的知识付费怎么做复购？
2. **错在哪里**
	：复购不是运营技巧，是用户完成一个问题后自然出现下一个问题
3. **隐含假设**
	：复购可以靠社群活跃和话术制造
4. **大白话改写**
	：用户解决当前问题后，还会遇到什么更高阶问题？
5. **真正该做的事**
	：设计问题阶梯：入门、执行、反馈、进阶、陪跑
6. **一句话**
	：复购来自真实进阶，不来自群里热闹

### 039\. 我可以教别人赚钱吗？

1. **原问题**
	：我可以教别人赚钱吗？
2. **错在哪里**
	：「教赚钱」太大，也最容易变成假承诺
3. **隐含假设**
	：赚钱可以被抽象方法直接传授
4. **大白话改写**
	：我能不能教某类人在某个场景下完成某个赚钱动作？
5. **真正该做的事**
	：把承诺缩小到可验证动作：找对标、设计产品、写销售页、做成交
6. **一句话**
	：不要教赚钱，教一个具体赚钱动作

### 040\. 我卖课没人买，是不是需求不够？

1. **原问题**
	：我卖课没人买，是不是需求不够？
2. **错在哪里**
	：没人买可能是需求、信任、定价、表达、客群、时机、渠道任何一环的问题
3. **隐含假设**
	：成交失败可以直接归因于需求
4. **大白话改写**
	：用户不买，是不需要、不相信、不着急，还是看不懂？
5. **真正该做的事**
	：逐环检查：流量来源、痛点表达、承诺、证据、价格、支付路径
6. **一句话**
	：没人买不是一个原因，是一条链上某个环节断了

## 五、自媒体平台

### 041\. 小红书现在还能不能做？

1. **原问题**
	：小红书现在还能不能做？
2. **错在哪里**
	：平台能不能做不是问题，具体账号、内容、产品、转化能不能跑通才是问题
3. **隐含假设**
	：平台整体机会决定个体结果
4. **大白话改写**
	：我这类产品能不能在小红书找到愿意停留和付款的人？
5. **真正该做的事**
	：找同类账号，看他们的内容形态、引流路径、产品价格和成交证据
6. **一句话**
	：平台从来不是不能做，是你的打法可能不能做

### 042\. 抖音是不是不适合知识博主？

1. **原问题**
	：抖音是不是不适合知识博主？
2. **错在哪里**
	：「知识博主」太宽，抖音也不是单一场景
3. **隐含假设**
	：平台有固定适配人群
4. **大白话改写**
	：我的知识能不能被短时间感知，并进入直播、私域或产品转化？
5. **真正该做的事**
	：测试短视频钩子、直播承接、低价产品或咨询转化
6. **一句话**
	：不是抖音不适合知识，是不适合慢吞吞的知识表达

### 043\. 公众号还有没有价值？

1. **原问题**
	：公众号还有没有价值？
2. **错在哪里**
	：价值不是平台属性，价值来自用户关系和内容深度
3. **隐含假设**
	：平台衰退就没有商业价值
4. **大白话改写**
	：我有没有需要长逻辑、强信任、可沉淀的内容？
5. **真正该做的事**
	：把公众号当深度信任资产，而不是涨粉工具
6. **一句话**
	：公众号不一定适合起量，但适合让人认真相信你

### 044\. 我应该全平台分发吗？

1. **原问题**
	：我应该全平台分发吗？
2. **错在哪里**
	：全平台可能是效率，也可能是用忙碌替代判断
3. **隐含假设**
	：分发越多，结果越好
4. **大白话改写**
	：同一内容在不同平台是否有适配后的有效表达？
5. **真正该做的事**
	：先跑通一个主平台，再把内容改写成适合其它平台的版本
6. **一句话**
	：全平台不是复制粘贴，是同一判断的多种表达

### 045\. 我账号被限流了吗？

1. **原问题**
	：我账号被限流了吗？
2. **错在哪里**
	：很多人用限流解释内容差，省掉复盘
3. **隐含假设**
	：数据差主要是平台压制
4. **大白话改写**
	：有没有证据证明平台异常，而不是内容没人想看？
5. **真正该做的事**
	：换题、换封面、换开头做对照测试，别先怪平台
6. **一句话**
	：限流可能存在，但它经常被用来保护自尊

### 046\. 我需要做私域吗？

1. **原问题**
	：我需要做私域吗？
2. **错在哪里**
	：私域不是微信群，不是朋友圈，也不是囤人头
3. **隐含假设**
	：把人加到微信就拥有了私域
4. **大白话改写**
	：我有没有能力持续影响这些人的购买决策？
5. **真正该做的事**
	：设计引流理由、承接内容、信任机制、成交路径和复购机制
6. **一句话**
	：私域不是联系人列表，是可反复触达并产生交易的关系资产

### 047\. 我该不该投流？

1. **原问题**
	：我该不该投流？
2. **错在哪里**
	：投流是放大器，不是救命药
3. **隐含假设**
	：自然流量不行，可以靠投流解决
4. **大白话改写**
	：我的内容和产品已经能在小规模流量下稳定转化吗？
5. **真正该做的事**
	：先算转化率、客单价、毛利和可承受获客成本，再小额测试
6. **一句话**
	：没跑通转化就投流，是花钱让问题更快暴露

### 048\. 我账号要不要垂直？

1. **原问题**
	：我账号要不要垂直？
2. **错在哪里**
	：「垂直」常被误解成只能讲一个窄话题，真正重要的是用户记忆一致
3. **隐含假设**
	：内容品类越窄，账号越容易成功
4. **大白话改写**
	：别人能不能用一句话记住我持续解决哪类问题？
5. **真正该做的事**
	：统一目标用户、核心问题和判断风格，允许形式变化
6. **一句话**
	：垂直不是题材单一，是用户记忆稳定

### 049\. 我的视频完播率低怎么办？

1. **原问题**
	：我的视频完播率低怎么办？
2. **错在哪里**
	：完播率低不是一个问题，是开头、节奏、信息密度、承诺兑现的问题
3. **隐含假设**
	：有一个技巧能直接提高完播率
4. **大白话改写**
	：用户为什么要从第一秒看到最后一秒？
5. **真正该做的事**
	：前 3 秒给冲突，中段每 10 秒推进，结尾兑现开头承诺
6. **一句话**
	：完播率不是求来的，是每一秒都给用户继续看的理由

### 050\. 我发哪个时间段最好？

1. **原问题**
	：我发哪个时间段最好？
2. **错在哪里**
	：发布时间通常不是核心变量，尤其在内容本身没过关时
3. **隐含假设**
	：选择时间能显著改变内容命运
4. **大白话改写**
	：我的内容是否强到换个时间也有人看？
5. **真正该做的事**
	：先优化选题、标题、封面、开头，再做发布时间测试
6. **一句话**
	：烂内容不会因为晚上八点发布就变好

## 六、执行力 / 心理

### 051\. 我知道该做什么，但就是不做怎么办？

1. **原问题**
	：我知道该做什么，但就是不做怎么办？
2. **错在哪里**
	：这不是方法问题，是行动意愿和失败风险的问题
3. **隐含假设**
	：我缺一个更好的方法来启动
4. **大白话改写**
	：我是不是在用「准备」保护自己不被结果评价？
5. **真正该做的事**
	：把任务缩小到今天必须完成、能被外界看见的一步
6. **一句话**
	：你不是不知道怎么做，你是不想面对做了也可能不行

### 052\. 我总是拖延怎么办？

1. **原问题**
	：我总是拖延怎么办？
2. **错在哪里**
	：拖延不是原因，是结果。它通常有目的：保护自我评价
3. **隐含假设**
	：拖延是一种坏习惯，只要改习惯就行
4. **大白话改写**
	：拖延帮我逃避了什么评价、失败或责任？
5. **真正该做的事**
	：公开一个小承诺，设定截止时间，让结果进入现实
6. **一句话**
	：拖延不是时间管理问题，是自尊保护机制

### 053\. 我想赚钱但没有动力怎么办？

1. **原问题**
	：我想赚钱但没有动力怎么办？
2. **错在哪里**
	：「想赚钱」不能用语言定义，要用行动定义
3. **隐含假设**
	：我确实想赚钱，只是动力不够
4. **大白话改写**
	：我今天有没有做任何能直接带来收入的动作？
5. **真正该做的事**
	：每天做一个离钱最近的动作：报价、销售、成交、交付、复购
6. **一句话**
	：想赚钱的意思就是不想赚钱，正在赚钱才是想赚钱

### 054\. 我怕失败，所以不敢开始怎么办？

1. **原问题**
	：我怕失败，所以不敢开始怎么办？
2. **错在哪里**
	：怕失败不是问题，所有人都怕。问题是你把怕失败当成不行动的许可
3. **隐含假设**
	：不害怕了才能开始
4. **大白话改写**
	：我能不能在害怕的同时完成一个低成本测试？
5. **真正该做的事**
	：把失败成本降到可承受，然后立刻测试
6. **一句话**
	：勇敢不是不怕，是害怕也把付款链接发出去

### 055\. 我总想换方向怎么办？

1. **原问题**
	：我总想换方向怎么办？
2. **错在哪里**
	：可能不是方向错，是你每次快进入真实摩擦就逃走
3. **隐含假设**
	：下一个方向会更适合我
4. **大白话改写**
	：我是在寻找方向，还是在逃避深入？
5. **真正该做的事**
	：给当前方向设定一个不可提前退出的验证周期和指标
6. **一句话**
	：频繁换方向的人，不是在找路，是在躲坡

### 056\. 我准备好了再开始可以吗？

1. **原问题**
	：我准备好了再开始可以吗？
2. **错在哪里**
	：「准备好」没有终点，常常是延迟行动的体面说法
3. **隐含假设**
	：充分准备可以消除失败
4. **大白话改写**
	：我现在缺的准备，是否真的会影响第一次测试？
5. **真正该做的事**
	：区分必要准备和心理准备，只完成必要准备
6. **一句话**
	：准备如果不通向行动，就是拖延的精装修

### 057\. 我执行力差，是不是不适合创业？

1. **原问题**
	：我执行力差，是不是不适合创业？
2. **错在哪里**
	：执行力差可能是商业模式太差、任务太虚、反馈太远，也可能是心理逃避
3. **隐含假设**
	：执行力是固定人格属性
4. **大白话改写**
	：我为什么不执行：看不到钱、不会做、怕失败，还是不相信这事？
5. **真正该做的事**
	：把任务改成明确、短周期、有反馈、离钱近的动作
6. **一句话**
	：执行力不是人格光环，是任务设计和风险承受的结果

### 058\. 我太焦虑了，怎么缓解？

1. **原问题**
	：我太焦虑了，怎么缓解？
2. **错在哪里**
	：如果焦虑来自业务不确定性，缓解情绪不能替代处理现实
3. **隐含假设**
	：情绪好了，问题就能解决
4. **大白话改写**
	：我的焦虑对应哪个未处理的现实问题？
5. **真正该做的事**
	：列出最小可控动作，今天完成一个现实推进
6. **一句话**
	：能用行动处理的焦虑，不要全交给情绪管理

### 059\. 我想做但家人不支持怎么办？

1. **原问题**
	：我想做但家人不支持怎么办？
2. **错在哪里**
	：把家人不支持当成不能做的原因，可能是在外包责任
3. **隐含假设**
	：得到支持才有资格开始
4. **大白话改写**
	：在不影响基本责任的前提下，我能不能小成本验证？
5. **真正该做的事**
	：用业余时间做低风险测试，用结果而不是争论换取空间
6. **一句话**
	：家人不支持不是商业障碍，除非你的商业计划需要他们付款

### 060\. 我是不是应该先学习一段时间？

1. **原问题**
	：我是不是应该先学习一段时间？
2. **错在哪里**
	：学习可能是必要动作，也可能是行动替代品
3. **隐含假设**
	：学得更多就更容易开始
4. **大白话改写**
	：我学的东西有没有立刻用于一个真实任务？
5. **真正该做的事**
	：学习和行动绑定：学 1 小时，马上产出一个可发布、可销售、可验证的东西
6. **一句话**
	：不能进入行动的学习，只是在购买进步幻觉

## 七、对标 / 模仿

### 061\. 我该模仿谁？

1. **原问题**
	：我该模仿谁？
2. **错在哪里**
	：不能按喜欢、气质、审美选对标，要按赚钱、看懂、能仿选
3. **隐含假设**
	：对标应该和我像
4. **大白话改写**
	：谁已经用我能复制的方式赚到了我想赚的钱？
5. **真正该做的事**
	：用五重过滤：他赚钱吗、看懂吗、能仿吗、排除自我吗、不讨论本质吗
6. **一句话**
	：找对标不是找偶像，是找作业答案

### 062\. 我不想模仿别人，想做自己

1. **原问题**
	：我不想模仿别人，想做自己
2. **错在哪里**
	：0 到 1 阶段，「做自己」常常是逃避模仿难度
3. **隐含假设**
	：原创比模仿更高级
4. **大白话改写**
	：我是不是不愿意承认别人已经把答案做出来了？
5. **真正该做的事**
	：先完整复制一个赚钱模型，再谈差异化
6. **一句话**
	：先活下来，才有资格做自己

### 063\. 我的对标太强了，模仿不了怎么办？

1. **原问题**
	：我的对标太强了，模仿不了怎么办？
2. **错在哪里**
	：对标不是崇拜对象，是可拆解对象。不能仿就换更近的
3. **隐含假设**
	：对标必须是行业头部
4. **大白话改写**
	：我能不能找到一个比我强 3 倍、不是强 300 倍的人？
5. **真正该做的事**
	：找阶段相近、路径清楚、资源可复制的中腰部对标
6. **一句话**
	：对标太远不是灯塔，是幻觉

### 064\. 我已经模仿了，为什么没效果？

1. **原问题**
	：我已经模仿了，为什么没效果？
2. **错在哪里**
	：很多「模仿」只是模仿表面，没有模仿价格、节奏、渠道、话术、交付
3. **隐含假设**
	：看起来像就叫模仿
4. **大白话改写**
	：我和对标在哪些关键环节不一致？
5. **真正该做的事**
	：逐项对比产品、价格、封面、标题、频率、私域、销售、交付
6. **一句话**
	：模仿不是学个皮肤，是复制一整套赚钱动作

### 065\. 对标的业务我不喜欢怎么办？

1. **原问题**
	：对标的业务我不喜欢怎么办？
2. **错在哪里**
	：喜欢不是商业筛选标准，利润和可执行才是
3. **隐含假设**
	：喜欢才能做好
4. **大白话改写**
	：这件事具体哪一步我做不了，还是只是我不想做？
5. **真正该做的事**
	：如果能做且赚钱，先做验证；如果关键步骤真的不能做，换对标
6. **一句话**
	：兴趣不是入场券，成交才是

### 066\. 我想做差异化，怎么做？

1. **原问题**
	：我想做差异化，怎么做？
2. **错在哪里**
	：没有同质化能力就谈差异化，通常是在逃避基本功
3. **隐含假设**
	：不同就更有价值
4. **大白话改写**
	：我是否已经能做到行业平均水平，再在某一点更好？
5. **真正该做的事**
	：先复制对标的基本盘，再在用户、交付、表达或价格上微调
6. **一句话**
	：差异化不是为了不同，是为了在同类里更容易成交

### 067\. 我找不到对标怎么办？

1. **原问题**
	：我找不到对标怎么办？
2. **错在哪里**
	：找不到可能不是没有，是你的关键词、场景、产品定义不清楚
3. **隐含假设**
	：对标应该和我的想法完全一样
4. **大白话改写**
	：有没有人在服务同类客户、解决相似问题、使用类似交付？
5. **真正该做的事**
	：按客户、问题、价格、交付、渠道五个维度扩展搜索
6. **一句话**
	：对标不是找双胞胎，是找可复制的赚钱结构

### 068\. 我能不能模仿国外案例？

1. **原问题**
	：我能不能模仿国外案例？
2. **错在哪里**
	：关键不是国外国内，而是买家、渠道、支付习惯、交付条件是否可迁移
3. **隐含假设**
	：国外成功的东西国内也会成功
4. **大白话改写**
	：这个模型迁移后，哪些对象和关系会变？
5. **真正该做的事**
	：逐项检查用户、价格、渠道、信任、监管、交付成本
6. **一句话**
	：跨市场模仿，模仿的是结构，不是包装

### 069\. 我应该模仿大号还是小号？

1. **原问题**
	：我应该模仿大号还是小号？
2. **错在哪里**
	：粉丝量不是唯一标准，赚钱能力和路径可见性更重要
3. **隐含假设**
	：粉丝越多越值得模仿
4. **大白话改写**
	：哪个账号的商业链条最清楚、最可复制？
5. **真正该做的事**
	：优先找粉丝不夸张但产品明确、转化路径清楚、交付稳定的账号
6. **一句话**
	：大号给你热闹，小号可能给你答案

### 070\. 我模仿会不会被骂抄袭？

1. **原问题**
	：我模仿会不会被骂抄袭？
2. **错在哪里**
	：把模仿商业结构和抄袭具体表达混在一起
3. **隐含假设**
	：模仿就是不道德
4. **大白话改写**
	：我是在复制赚钱逻辑，还是在搬运别人的具体作品？
5. **真正该做的事**
	：模仿结构、流程、节奏和策略，重新生成自己的表达和交付
6. **一句话**
	：抄内容是懒，仿结构是学习

## 八、产品 / 定价

### 071\. 我的产品很好，为什么卖不出去？

1. **原问题**
	：我的产品很好，为什么卖不出去？
2. **错在哪里**
	：「好」是你的评价，买家不付款就说明好没有被交易验证
3. **隐含假设**
	：产品好应该自动卖得出去
4. **大白话改写**
	：买家是否理解它、相信它、急需它、愿意为它付这个价格？
5. **真正该做的事**
	：访谈未购买用户，检查需求、表达、信任、价格和购买路径
6. **一句话**
	：卖不出去的「好产品」，至少在交易意义上还不够好

### 072\. 我要不要做免费版？

1. **原问题**
	：我要不要做免费版？
2. **错在哪里**
	：免费版不是善意，是获客、筛选、教育或试用策略
3. **隐含假设**
	：免费能降低用户决策门槛，所以一定有利
4. **大白话改写**
	：免费版会带来付费转化，还是只会吸引不付费用户？
5. **真正该做的事**
	：明确免费版的边界、升级理由和转化路径
6. **一句话**
	：免费的东西如果不能通向付费，就是成本中心

### 073\. 我的产品应该做多功能还是单功能？

1. **原问题**
	：我的产品应该做多功能还是单功能？
2. **错在哪里**
	：功能多少不是问题，用户任务完成度才是问题
3. **隐含假设**
	：功能越多，价值越高
4. **大白话改写**
	：用户购买这个产品最想完成的一个关键任务是什么？
5. **真正该做的事**
	：先把核心任务做到可验证，再增加辅助功能
6. **一句话**
	：功能不是价值，完成任务才是价值

### 074\. 我该不该降价？

1. **原问题**
	：我该不该降价？
2. **错在哪里**
	：降价不能解决价值表达、信任和客群不匹配的问题
3. **隐含假设**
	：用户不买主要是嫌贵
4. **大白话改写**
	：用户觉得贵，是价格高，还是价值没被证明？
5. **真正该做的事**
	：先检查证据、承诺、痛点和竞品价格，再决定是否改产品层级
6. **一句话**
	：客户说贵，很多时候是在说「不信」

### 075\. 我想做低价引流款，可以吗？

1. **原问题**
	：我想做低价引流款，可以吗？
2. **错在哪里**
	：低价引流款必须服务利润款，否则只是卖便宜货
3. **隐含假设**
	：低价可以自然带来后续成交
4. **大白话改写**
	：买完低价产品的人，为什么会继续买高价产品？
5. **真正该做的事**
	：设计引流款和利润款的 10 倍价差、同一问题链和自然升级路径
6. **一句话**
	：引流款不是低价产品，是利润款的入口

### 076\. 我应该按成本定价还是按价值定价？

1. **原问题**
	：我应该按成本定价还是按价值定价？
2. **错在哪里**
	：成本决定你能不能活，价值决定用户愿不愿意买，竞品决定市场参照
3. **隐含假设**
	：存在一种单一正确的定价方法
4. **大白话改写**
	：这个价格能不能覆盖成本、匹配用户价值感，并在竞品中有位置？
5. **真正该做的事**
	：同时看成本底线、用户收益、竞品价格和转化数据
6. **一句话**
	：定价不是算术题，是交易结构题

### 077\. 我的产品要不要做会员制？

1. **原问题**
	：我的产品要不要做会员制？
2. **错在哪里**
	：会员制不是收费形式，是持续价值交付能力
3. **隐含假设**
	：订阅能让收入更稳定
4. **大白话改写**
	：用户每个月为什么还愿意继续付钱？
5. **真正该做的事**
	：明确持续内容、服务、工具、社群或权益的真实价值
6. **一句话**
	：不能持续创造价值的会员制，只是分期收割

### 078\. 我需要先做 MVP 吗？

1. **原问题**
	：我需要先做 MVP 吗？
2. **错在哪里**
	：很多人把 MVP 理解成粗糙产品，但 MVP 是最小验证，不是最小敷衍
3. **隐含假设**
	：做一个简陋版本就算验证
4. **大白话改写**
	：我最需要验证的核心风险是什么？
5. **真正该做的事**
	：只做能验证付款、使用或留存的最小交付
6. **一句话**
	：MVP 不是少做功能，是精准验证最大风险

### 079\. 用户说想要，我要不要做？

1. **原问题**
	：用户说想要，我要不要做？
2. **错在哪里**
	：用户说想要不等于愿意付款、愿意使用、愿意迁移
3. **隐含假设**
	：口头需求等于真实需求
4. **大白话改写**
	：用户愿不愿意现在付钱、付时间或付迁移成本？
5. **真正该做的事**
	：用预售、排队、定金、试用行为验证需求
6. **一句话**
	：嘴上的需求很便宜，付款的需求才有重量

### 080\. 我该不该做定制化？

1. **原问题**
	：我该不该做定制化？
2. **错在哪里**
	：定制化可能是高利润入口，也可能毁掉标准化和交付效率
3. **隐含假设**
	：客户愿意付钱，定制就值得做
4. **大白话改写**
	：这次定制能不能沉淀成 SOP、案例或更高价格产品？
5. **真正该做的事**
	：只接能赚钱、能学习、能沉淀的定制，不能沉淀就高价
6. **一句话**
	：定制化可以接，但不要用团队生命给客户做手工艺

## 九、流量 / 私域

### 081\. 我怎么获得精准流量？

1. **原问题**
	：我怎么获得精准流量？
2. **错在哪里**
	：「精准流量」是模糊词。精准是对谁、对什么产品、对哪个转化动作精准？
3. **隐含假设**
	：存在一种天然精准的人群
4. **大白话改写**
	：哪些人最可能因为哪个问题购买我的产品？
5. **真正该做的事**
	：定义购买场景，围绕购买前的问题生产内容和投放
6. **一句话**
	：精准流量不是标签精准，是付费意图精准

### 082\. 我粉丝少，能赚钱吗？

1. **原问题**
	：我粉丝少，能赚钱吗？
2. **错在哪里**
	：粉丝量和收入不是线性关系
3. **隐含假设**
	：粉丝多才有资格赚钱
4. **大白话改写**
	：我有没有少量但高信任、高需求、高支付能力的人？
5. **真正该做的事**
	：先做高客单或强需求产品，用成交验证，而不是等涨粉
6. **一句话**
	：粉丝少不是问题，粉丝不买才是问题

### 083\. 我流量大但不赚钱怎么办？

1. **原问题**
	：我流量大但不赚钱怎么办？
2. **错在哪里**
	：流量不是收入。问题在买家匹配、产品设计、转化路径或信任机制
3. **隐含假设**
	：有流量就应该有收入
4. **大白话改写**
	：这些看我的人里面，有没有我的买家？
5. **真正该做的事**
	：检查受众画像、内容承诺、产品入口、价格和成交链路
6. **一句话**
	：没有交易设计的流量，只是观众席

### 084\. 我怎么把公域导到私域？

1. **原问题**
	：我怎么把公域导到私域？
2. **错在哪里**
	：导流不是技术动作，是价值交换
3. **隐含假设**
	：用户会因为你让他加微信就加
4. **大白话改写**
	：用户为什么要离开平台，进入我的微信场景？
5. **真正该做的事**
	：提供清晰的私域理由：资料、诊断、服务、案例、工具或后续陪跑
6. **一句话**
	：导流不是把人搬家，是给他一个搬家的理由

### 085\. 我的私域怎么成交？

1. **原问题**
	：我的私域怎么成交？
2. **错在哪里**
	：私域成交不是群发话术，是信任、需求、时机和产品匹配
3. **隐含假设**
	：只要进了私域，就可以用话术成交
4. **大白话改写**
	：用户在私域里经历了什么，才会相信我能解决他的问题？
5. **真正该做的事**
	：设计内容教育、案例证明、需求识别、报价和跟进节奏
6. **一句话**
	：私域成交不是收割，是把信任推到付款那一步

### 086\. 我需要建社群吗？

1. **原问题**
	：我需要建社群吗？
2. **错在哪里**
	：社群不是把人拉进群，社群需要共同任务、规则和持续价值
3. **隐含假设**
	：有群就有粘性
4. **大白话改写**
	：这群人在一起能完成一个单独完成不了的任务吗？
5. **真正该做的事**
	：先定义社群目的、成员筛选、活动机制、交付和商业闭环
6. **一句话**
	：没有共同任务的社群，就是吵闹的通讯录

### 087\. 我怎么提高转化率？

1. **原问题**
	：我怎么提高转化率？
2. **错在哪里**
	：转化率不是独立按钮，它由流量质量、承诺、信任、价格、路径共同决定
3. **隐含假设**
	：转化率可以靠一个技巧优化
4. **大白话改写**
	：用户在哪一步从想买变成不买？
5. **真正该做的事**
	：拆漏斗：曝光、点击、咨询、报价、付款，每一步找流失原因
6. **一句话**
	：转化率不是玄学，是每一步少掉的人

### 088\. 我怎么做裂变？

1. **原问题**
	：我怎么做裂变？
2. **错在哪里**
	：裂变不是增长魔法，用户愿意传播需要利益、身份、关系或强价值
3. **隐含假设**
	：设计一个活动就能让用户帮我传播
4. **大白话改写**
	：用户转发我的东西，对他自己有什么好处？
5. **真正该做的事**
	：设计明确奖励、可传播话术、低门槛动作和产品承接
6. **一句话**
	：裂变不是让用户帮你，是让用户转发时也在帮自己

### 089\. 我要不要买流量？

1. **原问题**
	：我要不要买流量？
2. **错在哪里**
	：买流量的前提是你知道每个客户值多少钱
3. **隐含假设**
	：买来更多人，就会带来更多钱
4. **大白话改写**
	：我花多少钱买一个客户，能赚回多少钱？
5. **真正该做的事**
	：计算 CAC、客单价、毛利、复购、回本周期，再小额测试
6. **一句话**
	：不会算账就买流量，是把钱交给平台做公益

### 090\. 我内容引来的用户不精准怎么办？

1. **原问题**
	：我内容引来的用户不精准怎么办？
2. **错在哪里**
	：用户不精准往往是内容承诺不精准
3. **隐含假设**
	：平台把错误的人推给了我
4. **大白话改写**
	：我的内容吸引的是看热闹的人，还是有购买问题的人？
5. **真正该做的事**
	：调整选题、标题和案例，让内容只吸引目标买家觉得有用
6. **一句话**
	：你用热闹钓鱼，就别怪上来的不是客户

## 十、综合判断

### 091\. 我该辞职创业吗？

1. **原问题**
	：我该辞职创业吗？
2. **错在哪里**
	：辞职不是创业的前提，收入验证才是
3. **隐含假设**
	：只有辞职才能认真创业
4. **大白话改写**
	：我的副业收入、现金流和验证程度是否支持我承担全职风险？
5. **真正该做的事**
	：先用业余时间做到稳定成交，准备现金流，再决定是否辞职
6. **一句话**
	：辞职不是创业按钮，是风险放大器

### 092\. 我该找合伙人吗？

1. **原问题**
	：我该找合伙人吗？
2. **错在哪里**
	：很多人找合伙人不是因为需要互补，是因为自己不敢承担
3. **隐含假设**
	：有人一起做，成功概率更高
4. **大白话改写**
	：这个人能补上我无法补的关键能力，并承担明确责任吗？
5. **真正该做的事**
	：先定义股权、权责、投入、退出机制和不可替代贡献
6. **一句话**
	：合伙不是找人壮胆，是交换不可替代能力

### 093\. 我要不要融资？

1. **原问题**
	：我要不要融资？
2. **错在哪里**
	：融资不是成功，是一种增长工具和约束
3. **隐含假设**
	：融到钱代表项目更有价值
4. **大白话改写**
	：我的业务是否已经证明钱能有效放大增长？
5. **真正该做的事**
	：先验证单位经济模型，再判断外部资金是否能加速
6. **一句话**
	：不能用小钱赚钱，就不要急着拿大钱烧

### 094\. 我该不该转型？

1. **原问题**
	：我该不该转型？
2. **错在哪里**
	：「转型」太大。可能只是换产品、换客户、换渠道、换定价
3. **隐含假设**
	：当前问题需要整体推倒重来
4. **大白话改写**
	：现有业务到底是哪一环不成立？
5. **真正该做的事**
	：拆获客、转化、交付、利润、复购，找到断点再改
6. **一句话**
	：不要用转型掩盖你没拆清楚的问题

### 095\. 我是不是认知不够？

1. **原问题**
	：我是不是认知不够？
2. **错在哪里**
	：「认知」经常是一个高级垃圾桶，什么解释不了都往里面倒
3. **隐含假设**
	：赚不到钱是因为认知低
4. **大白话改写**
	：我具体缺哪个判断、信息、技能或行动？
5. **真正该做的事**
	：把认知问题翻译成可补的信息、可练的技能、可做的动作
6. **一句话**
	：别拿认知当烟雾弹，缺什么就说什么

### 096\. 我现在太迷茫了怎么办？

1. **原问题**
	：我现在太迷茫了怎么办？
2. **错在哪里**
	：迷茫通常不是信息不足，而是选项太多、标准不清、行动太少
3. **隐含假设**
	：想清楚以后才能开始
4. **大白话改写**
	：我当前最小的可验证选择是什么？
5. **真正该做的事**
	：选一个低成本方向，设定 14 天验证指标，用行动减少迷雾
6. **一句话**
	：迷茫不是靠想消失的，是被现实反馈冲淡的

### 097\. 我该相信直觉还是数据？

1. **原问题**
	：我该相信直觉还是数据？
2. **错在哪里**
	：直觉和数据不是二选一。直觉提出假设，数据校验假设
3. **隐含假设**
	：两者必然冲突
4. **大白话改写**
	：我的直觉能不能被低成本测试验证？
5. **真正该做的事**
	：把直觉写成可测试命题，收集真实行为数据
6. **一句话**
	：未经验证的直觉叫猜，能设计验证的直觉叫判断

### 098\. 我应该坚持还是放弃？

1. **原问题**
	：我应该坚持还是放弃？
2. **错在哪里**
	：坚持和放弃都太抽象。关键是指标有没有改善，假设有没有被证伪
3. **隐含假设**
	：坚持本身有价值，或者放弃本身是失败
4. **大白话改写**
	：我继续做下去，是因为数据在变好，还是因为不甘心？
5. **真正该做的事**
	：设定退出标准：时间、投入、关键指标、验证结果
6. **一句话**
	：没有指标的坚持，是情绪；有证据的放弃，是理性

### 099\. 我怎么找到自己的优势？

1. **原问题**
	：我怎么找到自己的优势？
2. **错在哪里**
	：优势不是自我想象，是在竞争和交易中被证明的相对能力
3. **隐含假设**
	：优势可以通过内省找到
4. **大白话改写**
	：别人在哪些事情上愿意持续找我、信我、付钱给我？
5. **真正该做的事**
	：做 10 个真实交付，观察哪类问题你解决得更快、更稳、更被认可
6. **一句话**
	：优势不是你觉得自己擅长什么，是市场反复把什么交给你

### 100\. 我下一步该做什么？

1. **原问题**
	：我下一步该做什么？
2. **错在哪里**
	：如果没有目标、现状和约束，下一步无法被判断
3. **隐含假设**
	：别人可以脱离上下文告诉我一个正确动作
4. **大白话改写**
	：我现在最接近钱、反馈或关键验证的一步是什么？
5. **真正该做的事**
	：列出当前目标、已有资源、最大风险，然后选择能验证最大风险的最小动作
6. **一句话**
	：下一步不是最舒服的一步，是最能让现实回答你的一步
